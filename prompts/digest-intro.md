# 摘要格式提示词

你的任务是将收集到的 API 信息组织成一份易读的每日摘要。

## 整体结构

```
# API Hunter 每日摘要
{今天的日期}

---

## 🔥 开发者推荐

{来自追踪账号的 API 推荐}

---

## 🔍 热门发现

{来自关键词搜索的 API}

---

## 📊 今日统计

- 追踪账号: {数量} 个
- 开发者推文: {数量} 条
- 搜索发现: {数量} 条
- 总计: {数量} 个 API

---

💡 想要调整摘要内容？回复告诉我你的偏好。
```

## 开发者推荐部分

按账号分组，每个账号下列出他们分享的 API：

```
### @levelsio (Pieter Levels)

**Resend** - 开发者友好的邮件发送 API
- **用途**: 在应用中发送事务性邮件
- **特点**: 免费额度 100 封/天，简单的 SDK
- **链接**: https://resend.com

**Supabase** - 开源的 Firebase 替代品
- **用途**: 后端即服务（数据库、认证、存储）
- **特点**: 完全开源，基于 PostgreSQL
- **链接**: https://supabase.com
```

## 热门发现部分

不分组，直接列出搜索发现的 API：

## 语言

- 如果 config.language 是 "zh"，使用中文
- 如果是 "en"，使用英文
- 如果是 "bilingual"，中英文对照（中文在前，英文在后）

## 注意事项

1. **保持简洁**: 每个 API 的描述控制在 3-4 行
2. **突出重点**: 用粗体标注 API 名称和关键信息
3. **保留链接**: 所有链接必须保留
4. **分类清晰**: 开发者推荐和热门发现分开
5. **统计准确**: 最后的统计数据要准确

## 如果内容为空

如果某个部分没有内容，显示：

```
## 🔥 开发者推荐

今天没有新的推荐。明天再来看看吧！
```
