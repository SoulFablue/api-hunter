# 内容质量过滤提示词

你的任务是判断一条推文是否值得包含在 API 摘要中。

## 保留的内容（输出 keep: true）

✅ **真实的产品发布**
- 有官网链接
- 有具体的功能描述
- 来自可信的开发者

✅ **真实的使用体验**
- 开发者分享实际使用心得
- 有具体的用例或场景
- 提到了优缺点

✅ **有价值的推荐**
- 解决了实际问题
- 有一定热度（点赞 > 5）
- 包含有用的信息

## 过滤掉的内容（输出 keep: false）

❌ **纯广告**
- "限时优惠"、"立即购买"、"折扣码"
- 过度营销的语言
- 没有实质内容

❌ **低质量内容**
- 只有链接，没有描述
- 信息不完整
- 互动数据太低（点赞 < 5）

❌ **垃圾信息**
- 诈骗、钓鱼链接
- 无关内容
- 重复的转发

## 输出格式

对于每条推文，输出 JSON：

```json
{
  "keep": true,
  "reason": "真实的产品发布，有详细功能描述和官网链接",
  "confidence": "high"
}
```

或

```json
{
  "keep": false,
  "reason": "纯广告，包含'限时优惠'等营销语言",
  "confidence": "high"
}
```

## 判断标准

1. **有链接** - 必须有官网或文档链接
2. **有描述** - 必须说明 API 做什么
3. **有热度** - 至少 5 个赞（说明有人认可）
4. **非广告** - 不是纯粹的营销推广
5. **可信来源** - 来自真实的开发者账号

## 边界情况

- 如果不确定，倾向于 **保留**（keep: true），让用户自己判断
- 如果明显是垃圾/广告，果断 **过滤**（keep: false）
